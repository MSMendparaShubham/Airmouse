Overview

About AirMouse

AirMouse is a high-performance ICT project utilizing a hardware-based approach to spatial computing. By integrating an MPU6050 6-axis motion tracking sensor with an ESP-series microcontroller, it translates three-dimensional hand tilts and movements into precise desktop mouse actions. Unlike camera-based systems, AirMouse provides low-latency, surface-independent control that works in any lighting condition, offering a professional solution for presentations and ergonomic desktop navigation.

Mechanism

How It Works

01. Motion Sensing

The onboard MPU6050 sensor detects minute changes in gravity and acceleration, capturing precise 6-axis data from your hand.

02. Data Processing

The ESP microcontroller processes high-speed sensor data, filtering out noise to reveal clean directional vectors for the cursor.

03. Wireless Transmission

Movement data is transmitted wirelessly to the PC over Wi-Fi, Bluetooth, or through a dedicated 2.4GHz USB receiver.

04. Desktop Integration

The ESP32 sends motion data to the PC via Bluetooth, Wi-Fi, or USB Serial. A lightweight driver converts sensor values into cursor movement, while the physical switch triggers click actions — the PC recognises it as a standard HID mouse input.

Development

Source Code

ESP32 + MPU6050 AirMouse Firmware for Arduino IDE.

airmouse.ino
#include <Wire.h>
#include <Adafruit_MPU6050.h>
#include <Adafruit_Sensor.h>
#include <BleMouse.h>

Adafruit_MPU6050 mpu;
BleMouse bleMouse("ESP32 Air Mouse", "GSN Creation", 100);

const int leftButtonPin  = 25;  
const int rightButtonPin = 26;

void setup() {
  Serial.begin(115200);

  pinMode(leftButtonPin, INPUT_PULLUP);
  pinMode(rightButtonPin, INPUT_PULLUP);

  Wire.begin();
  if (!mpu.begin()) {
    Serial.println("MPU6050 not found!");
    while (1);
  }

  bleMouse.begin();

  // Configure sensitivity
  mpu.setAccelerometerRange(MPU6050_RANGE_4_G);
  mpu.setGyroRange(MPU6050_RANGE_500_DEG);
}

void loop() {
  sensors_event_t a, g, temp;
  mpu.getEvent(&a, &g, &temp);

  if (bleMouse.isConnected()) {
    // Gyro values give smooth pointer control
    int moveX = g.gyro.x * 8;   // adjust 8 → sensitivity
    int moveY = g.gyro.y * -8;

    bleMouse.move(moveX, moveY);

    // Left click
    if (!digitalRead(leftButtonPin)) {
      bleMouse.press(MOUSE_LEFT);
    } else {
      bleMouse.release(MOUSE_LEFT);
    }

    // Right click
    if (!digitalRead(rightButtonPin)) {
      bleMouse.press(MOUSE_RIGHT);
    } else {
      bleMouse.release(MOUSE_RIGHT);
    }
  }

  delay(10);  // Small delay to stabilize motion
}

An air mouse is an input device that lets you control a computer cursor by moving the device through the air instead of sliding it on a flat surface. This AirMouse uses an MPU6050 gyroscope + accelerometer sensor mounted on an ESP32 DevKit V1 to detect 3D hand motion and translate it into precise cursor movement — no surface, no camera required.

Yes. The AirMouse communicates as a standard Bluetooth HID mouse, so it works with any computer that supports Bluetooth — including Windows, macOS, and Linux — without installing extra drivers. It can also be connected via USB serial for direct communication depending on the firmware configuration.

The AirMouse is powered through the ESP32 DevKit V1's onboard USB port. You can power it from a USB power bank, a wall adapter, or directly from your PC's USB port. For a fully wireless build, a small Li-Po battery with a charging module can be integrated into the enclosure.

Absolutely. Because the firmware runs on the ESP32, you can reprogram it using the Arduino IDE to adjust sensitivity, dead zones, button mappings, and even add custom gestures. The MPU6050 data can be filtered and scaled to suit different use cases — from presentations to media control.

For many everyday tasks — presentations, media playback, casual browsing, and distance control — the AirMouse works great as a standalone input device. For pixel-perfect tasks like graphic design or detailed photo editing, a traditional mouse on a surface still offers higher precision.

The AirMouse is not optimised for high-precision competitive gaming. The MPU6050 sensor introduces a small amount of drift over time, and gyroscope-based aiming is less accurate than an optical gaming mouse on a surface. It can work for casual or turn-based games, but fast-paced FPS titles are better served by a traditional gaming mouse.

This AirMouse is a gyroscopic mouse — the terms are often used interchangeably. It uses the MPU6050's 3-axis gyroscope and 3-axis accelerometer to detect orientation changes in real time. The ESP32 reads this data over I2C, applies filtering, and converts the motion into cursor deltas sent wirelessly to the PC.

Yes — that is one of its biggest advantages. Because motion is tracked entirely by the MPU6050 sensor inside the device, no desk, mousepad, or flat surface is needed. You can use it while standing at a podium, sitting on a couch, or walking around a room.

Yes — presentations are one of the best use cases for this AirMouse. You can walk around the stage, point at the screen, and advance slides without being tethered to a laptop. The physical button on the device can be mapped to left-click or slide-advance, giving you full control from a distance.

The AirMouse supports multiple connection methods depending on the firmware: Bluetooth (using the BleMouse library — pair it like any BT mouse), Wi-Fi (send UDP packets to a Python listener on your PC), or USB Serial (plug the ESP32 directly into your computer and use a Python script to read movement data). Bluetooth is the most convenient for everyday use.

Start by checking the power supply and USB connection to the ESP32. If using Bluetooth, try unpairing and re-pairing the device. If the cursor drifts, the MPU6050 may need a moment to calibrate — keep the device still for 1–2 seconds after powering on. You can also re-upload the firmware via Arduino IDE to reset all settings to default.

Yes. When connected via Bluetooth HID, the AirMouse is recognised as a standard mouse by Windows, macOS, Linux, and ChromeOS — no drivers needed. For the USB Serial mode, a lightweight Python script handles the translation on the host computer, which also runs cross-platform.

The MPU6050 provides 16-bit resolution gyroscope and accelerometer readings, which translates to smooth and responsive cursor movement. It is slightly less precise than a high-DPI optical mouse on a surface due to natural hand tremor and sensor drift, but for UI navigation, presentations, and media control it is more than accurate enough.

When using Bluetooth, the ESP32 operates on the 2.4 GHz band and uses adaptive frequency hopping to avoid interference with Wi-Fi routers and other Bluetooth devices. In practice, interference is rarely an issue in typical home or office environments. The Wi-Fi mode uses standard 802.11 b/g/n, which also coexists well with other network devices.

Bluetooth range on the ESP32 is typically up to 10 metres (30 feet) in open space, which is ideal for classroom presentations and conference rooms. Wi-Fi mode can extend this further depending on your router's coverage. Walls and obstacles will reduce range, but for most indoor use cases the range is more than sufficient.

No line of sight is required. The AirMouse uses Bluetooth or Wi-Fi radio signals which pass through walls and obstacles. Motion is tracked entirely by the MPU6050 sensor inside the device itself — there is no external camera or receiver that needs to "see" your hand. You can even use it with the device in your pocket (though that's not recommended for precision control).
